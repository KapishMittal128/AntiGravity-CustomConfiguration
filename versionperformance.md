## Historical Summary

The v2.0-era system benchmark concluded that Antigravity was already strong at:
- bounded frontend and full-stack execution
- risk-aware routing before dangerous backend work
- architecture-first planning for non-trivial tasks
- keeping project memory on disk instead of in transient chat context

The same benchmark identified three structural weaknesses:
- too much reliance on prompt obedience instead of hard enforcement
- degraded reliability under very large, chaotic prompts
- drift risk when state files or support docs become stale
